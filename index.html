Inline Keyboard Button

Tombol yang muncul di dalam pesan (attached ke bubble). Tidak mengubah keyboard sistem. Dipakai lewat InlineKeyboardMarkup.

🤖
@RexTechBot
● online
Pilih aksi yang kamu mau 👇
14:23 ✓✓
💡 Klik tombol-tombol di atas untuk simulasi aksi. Inline button tidak mengubah keyboard sistem — user tetap bisa ketik normal.
🔗
URL
Buka link di browser/in-app browser Telegram.
url: "https://..."
📡
Callback
Kirim data ke bot tanpa bikin pesan. Paling umum dipakai.
callback_data: "str"
🔄
Switch Inline
Pindah ke mode inline query di chat lain.
switch_inline_query: ""
💳
Pay
Trigger pembayaran / Telegram Stars invoice.
pay: true
🎮
Game
Buka Telegram HTML5 Game.
callback_game: {}
🔐
Login URL
OAuth login pakai akun Telegram (Seamless Login).
login_url: {...}
🚀
Web App
Buka Telegram Mini App (TWA) dari pesan.
web_app: {url: "..."}
📋
Copy Text
Klik → copy teks ke clipboard user (Bot API 7.11+).
copy_text: {text: "..."}
JAVASCRIPT — node-telegram-bot-api
// npm i node-telegram-bot-api
const TelegramBot = require('node-telegram-bot-api');
const bot = new TelegramBot(TOKEN, { polling: true });

bot.sendMessage(chatId, 'Pilih aksi:', {
  reply_markup: {
    inline_keyboard: [
      // Baris 1: URL buttons
      [
        { text: '🔗 Website', url: 'https://rextech.id' },
        { text: '📄 Docs',    url: 'https://docs.rextech.id' },
      ],
      // Baris 2: Callback buttons
      [
        { text: '⚙️ Settings', callback_data: 'settings' },
        { text: '📊 Stats',    callback_data: 'stats' },
      ],
      // Baris 3: Switch inline
      [{ text: '🔄 Share', switch_inline_query: '' }],
      // Baris 4: Payment
      [{ text: '💳 Bayar', pay: true }],
      // Baris 5: Web App
      [{ text: '🚀 Mini App', web_app: { url: 'https://app.rextech.id' } }],
    ]
  }
});

// Handle callback
bot.on('callback_query', (query) => {
  const data = query.data;
  bot.answerCallbackQuery(query.id, { text: `Kamu klik: ${data}` });
  // lanjut logika...
});
PYTHON — python-telegram-bot
# pip install python-telegram-bot
from telegram import InlineKeyboardButton, InlineKeyboardMarkup
from telegram.ext import Application, CommandHandler, CallbackQueryHandler

async def start(update, context):
    keyboard = [
        # Baris 1: URL
        [
            InlineKeyboardButton("🔗 Website", url="https://rextech.id"),
            InlineKeyboardButton("📄 Docs",    url="https://docs.rextech.id"),
        ],
        # Baris 2: Callback
        [
            InlineKeyboardButton("⚙️ Settings", callback_data="settings"),
            InlineKeyboardButton("📊 Stats",    callback_data="stats"),
        ],
        # Baris 3: Web App
        [InlineKeyboardButton("🚀 Mini App", web_app=WebAppInfo(url="https://app.rextech.id"))],
    ]
    reply_markup = InlineKeyboardMarkup(keyboard)
    await update.message.reply_text("Pilih aksi:", reply_markup=reply_markup)

async def handle_cb(update, context):
    query = update.callback_query
    await query.answer(f"Kamu klik: {query.data}")

Reply Keyboard Button

Gantiin keyboard sistem di bawah layar. Klik = kirim teks ke chat persis seperti user ngetik. Pakai ReplyKeyboardMarkup.

🤖
@RexTechBot
● online
Pilih menu di bawah 👇
14:24 ✓✓
⚠️ Perhatian: Reply Keyboard menggantikan keyboard sistem. Gunakan resize_keyboard: true agar ukurannya pas, dan one_time_keyboard: true agar otomatis hilang setelah diklik.
📝
Text Button
Klik = kirim teks ke chat. Paling sederhana.
text: "Menu"
📱
Request Contact
Minta nomor HP user (ada dialog konfirmasi).
request_contact: true
📍
Request Location
Minta koordinat GPS (butuh izin lokasi).
request_location: true
📊
Request Poll
Buka dialog buat poll/quiz baru.
request_poll: {type: "quiz"}
👤
Request Users
User pilih akun Telegram dari kontaknya.
request_users: {id: 1}
💬
Request Chat
User pilih grup/channel yg dia admin-nya.
request_chat: {id: 1}
🚀
Web App
Buka Mini App dari reply keyboard.
web_app: {url: "..."}
JAVASCRIPT — node-telegram-bot-api
// Reply Keyboard — menggantikan keyboard sistem
bot.sendMessage(chatId, 'Pilih menu:', {
  reply_markup: {
    keyboard: [
      // Baris 1: text biasa
      ['📋 Menu', '🔍 Cari', '⚙️ Settings'],
      // Baris 2: request kontak & lokasi
      [
        { text: '📱 Kirim Kontak', request_contact: true },
        { text: '📍 Kirim Lokasi', request_location: true },
      ],
      // Baris 3: poll & web app
      [
        { text: '📊 Buat Poll', request_poll: { type: 'quiz' } },
        { text: '🚀 Web App',  web_app: { url: 'https://app.rextech.id' } },
      ],
    ],
    resize_keyboard:    true,  // ukuran pas dengan konten
    one_time_keyboard:  true,  // hilang setelah diklik
    input_field_placeholder: 'Pilih menu...',
  }
});

// Handle teks dari reply keyboard
bot.on('message', (msg) => {
  if (msg.text === '📋 Menu') {
    bot.sendMessage(msg.chat.id, 'Ini menu utama!');
  }
  // kontak: msg.contact
  // lokasi: msg.location
});
PYTHON — python-telegram-bot
from telegram import ReplyKeyboardMarkup, KeyboardButton, WebAppInfo

async def start(update, context):
    keyboard = [
        # Baris 1: text biasa
        ["📋 Menu", "🔍 Cari", "⚙️ Settings"],
        # Baris 2: kontak & lokasi
        [
            KeyboardButton("📱 Kirim Kontak", request_contact=True),
            KeyboardButton("📍 Kirim Lokasi", request_location=True),
        ],
        # Baris 3: poll & web app
        [
            KeyboardButton("📊 Buat Poll", request_poll=KeyboardButtonPollType()),
            KeyboardButton("🚀 Web App", web_app=WebAppInfo("https://app.rextech.id")),
        ],
    ]
    markup = ReplyKeyboardMarkup(
        keyboard,
        resize_keyboard=True,
        one_time_keyboard=True,
        input_field_placeholder="Pilih menu...",
    )
    await update.message.reply_text("Pilih menu:", reply_markup=markup)

Remove Keyboard

Bukan tombol visual — ini instruksi ke Telegram client untuk menghapus / menyembunyikan Reply Keyboard yang aktif. Pakai ReplyKeyboardRemove.

🗑️
ReplyKeyboardRemove
Kirim ini bersamaan pesan → keyboard langsung hilang dari layar user.
Field wajib: remove_keyboard: true
Optional: selective: true — hanya hapus untuk user tertentu di grup.
Kapan pakai? Setelah user selesai milih opsi dari Reply Keyboard, kirim Remove supaya keyboard bersih dan user bisa ketik normal lagi.
JAVASCRIPT / PYTHON
// ─── Node.js ────────────────────────────────
bot.sendMessage(chatId, 'Oke, keyboard dihapus!', {
  reply_markup: {
    remove_keyboard: true,
    // selective: true  // hanya hapus untuk user tertentu
  }
});

// ─── Python ─────────────────────────────────
from telegram import ReplyKeyboardRemove

await update.message.reply_text(
    "Keyboard dihapus!",
    reply_markup=ReplyKeyboardRemove(),
)

// ─── GramJS ─────────────────────────────────
await client.invoke(new Api.messages.SendMessage({
  peer: chatId,
  message: 'Keyboard dihapus!',
  replyMarkup: new Api.ReplyKeyboardHide({ selective: false }),
}));

Force Reply

Memaksa Telegram client user masuk mode reply otomatis ke pesan bot. Cocok buat wizard / form step-by-step. Pakai ForceReply.

🤖
@RexTechBot
● online
Masukkan nama lengkap kamu:
14:25 ✓✓
↩️
Reply ke @RexTechBot — Masukkan nama lengkap
✏️ Ketik balasan...
💡 Use case: Ideal untuk registrasi, input form, atau alur tanya-jawab multi-step. User tidak perlu ngetik /command — langsung reply.
JAVASCRIPT / PYTHON
// ─── Node.js ────────────────────────────────
bot.sendMessage(chatId, 'Masukkan nama lengkap kamu:', {
  reply_markup: {
    force_reply: true,
    input_field_placeholder: 'Nama lengkap...',
    selective: true, // hanya untuk user yg di-mention (grup)
  }
});

// Handle reply-nya
bot.on('message', (msg) => {
  if (msg.reply_to_message?.from.is_bot) {
    const nama = msg.text;
    bot.sendMessage(chatId, `Halo, ${nama}!`);
  }
});

// ─── Python ─────────────────────────────────
from telegram import ForceReply

await update.message.reply_text(
    "Masukkan nama lengkap kamu:",
    reply_markup=ForceReply(
        input_field_placeholder="Nama lengkap...",
        selective=True,
    ),
)

// ─── GramJS ─────────────────────────────────
await client.invoke(new Api.messages.SendMessage({
  peer: chatId,
  message: 'Masukkan nama lengkap kamu:',
  replyMarkup: new Api.ForceReply({
    singleUse: true,
    selective: false,
    placeholder: 'Nama lengkap...',
  }),
}));

Perbandingan Lengkap

Kapan pakai yang mana? Lihat tabel ini.

TIPE POSISI KLIK = ? UBAH KEYBOARD? TERBAIK UNTUK
InlineKeyboard Di dalam pesan Callback / URL / Aksi ✗ Tidak Menu, navigasi, aksi, pembayaran, Web App
ReplyKeyboard Bawah layar (keyboard) Kirim teks ke chat ✓ Ya (ganti) Pilihan cepat, kirim kontak/lokasi/poll
RemoveKeyboard — — ✓ Ya (hapus) Bersihkan Reply Keyboard setelah selesai
ForceReply Bawah layar (bar reply) Mode reply otomatis ✓ Ya (mode reply) Form wizard, tanya-jawab step-by-step
🔑 Rule of thumb: Pakai InlineKeyboard untuk hampir semua kasus — paling fleksibel dan tidak mengubah UX keyboard user. Pakai ReplyKeyboard hanya kalau butuh quick-input seperti nomor HP atau lokasi. ForceReply cocok buat wizard multi-step di grup agar hanya user tertentu yang diforce reply.
